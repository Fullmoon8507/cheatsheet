【Webアプリケーションの機能概要】
|--------------------------+----------------------------------------------------------------|
| 機能                     | 内容                                                           |
|--------------------------+----------------------------------------------------------------|
| ルーティング             | クライアントのリクエスト内容とサーバーの処理内容とを紐づける。 |
| テンプレート             | 動的にHTMLを生成する。                                         |
| HTTPプロトコルのサポート | HTTPリクエスト(GETやPOSTなど)やレスポンスをサポートしている。  |
| データベース操作とORM    | データベースへの操作やORMの処理を提供している。                |
| セキュリティ             | XSSやSQL Injectionなどのセキュリティなどを対応している。       |
| セッション、クッキー     | セッションとクッキーに対応している。                           |
| Ajax                     | Ajax（非同期処理）に対応している。                             |
| RESTful API              | RESTful APIの処理を提供している。                              |
|--------------------------+----------------------------------------------------------------|

****************************************************************************************************
* TCP/IPモデル
****************************************************************************************************
【OSI参照モデルとの比較】
|----------------------+--------------------+-------------------------|
| OSI参照モデル        | TCP/IPモデル       | 主なプロトコル          |
|----------------------+--------------------+-------------------------|
| アプリケーション層   | アプリケーション層 | HTTP/HTTPS              |
| プレゼンテーション層 |                    | TLS/SSL、SSH            |
| セッション層         |                    | POP3、SMTP、IMAPなど    |
|----------------------+--------------------+-------------------------|
| トランスポート層     | トランスポート層   | TCP、UDPなど            |
|----------------------+--------------------+-------------------------|
| ネットワーク層       | インターネット層   | IPなど                  |
|----------------------+--------------------+-------------------------|
| データリンク層       | ネットワーク       | イーサネット、Wi-Fiなど |
| 物理層               | インターフェース層 |                         |
|----------------------+--------------------+-------------------------|

【各層のヘッダの例】
|------------------------------+--------------------+------------------------------|
| 送信側のデータ構造           | 階層               | 受信側のデータ構造           |
|------------------------------+--------------------+------------------------------|
| データ                       | -                  | データ                       |
| データ,HTTP                  | アプリケーション層 | HTTP,データ                  |
| データ,HTTP,TCP              | トランスポート層   | TCP,HTTP,データ              |
| データ,HTTP,TCP,IP           | インターネット層   | IP,TCP,HTTP,データ           |
| FCS,データ,HTTP,TCP,IP,Ether | インターネット層   | Ether,IP,TCP,HTTP,データ,FCS |
|------------------------------+--------------------+------------------------------|
| (電気信号)                   | (ハードウェア)     | (電気信号)                   |
|------------------------------+--------------------+------------------------------|

****************************************************************************************************
* REST
****************************************************************************************************
【RESTの原則】
|------------------------------------------------------------------------------------+------|
| 原則                                                                               | 備考 |
|------------------------------------------------------------------------------------+------|
| ステートレスなクライアント／サーバプロトコル                                       |      |
| すべてのリソースに適用できる「よく定義された操作」のセット                         |      |
| リソースを一意に識別する「汎用的な構文」                                           |      |
| アプリケーションの情報と状態遷移の両方を扱うことができる「ハイパーメディアの使用」 |      |
|------------------------------------------------------------------------------------+------|
